# Contributing

Thanks for your interest in improving this project. Contributions, bug reports, and practical suggestions are welcome.

## How to Contribute

1. Fork the repository.
2. Create a focused branch for your change.
3. Make the smallest useful change you can.
4. Test the project manually or with the available scripts.
5. Open a pull request with a clear explanation of what changed and why.

## Good Pull Requests

- Keep changes focused on one problem.
- Include screenshots for UI changes when helpful.
- Mention setup steps, tradeoffs, or known limitations.
- Do not include secrets, credentials, generated junk files, or unrelated formatting churn.

## Issues

When opening an issue, include:

- What you expected to happen.
- What actually happened.
- Steps to reproduce the problem.
- Screenshots, logs, or environment details when relevant.
